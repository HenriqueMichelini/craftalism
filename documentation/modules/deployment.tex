\bigskip

\section{Overview}

The Deployment repository contains the Docker Compose configuration and environment templates needed to run the entire Craftalism platform. It contains no application source code; all services are pulled from pre-built images published to the GitHub Container Registry. A single \code{docker compose up} command starts every service in the correct dependency order.

\section{Responsibilities}

\begin{itemize}
  \item Defines the full service topology and inter-service networking in \code{docker-compose.yml}.
  \item Manages a one-shot database initializer that creates the \code{authserver} database before the Authorization Server starts.
  \item Downloads and installs the Craftalism Economy plugin JAR into the Minecraft server container at startup using the configured \code{ECONOMY\_VERSION}.
  \item Templates an \code{env.example} file covering all required and optional configuration for the full stack.
  \item Exposes services on defined host ports for external access.
\end{itemize}

\subsection{Startup Dependency Order}

\begin{enumerate}
  \item \code{craftalism-postgres} starts and passes its health check.
  \item \code{auth-db-init} runs and creates the \code{authserver} database, then exits.
  \item \code{craftalism-auth-server} and \code{craftalism-api} start in parallel (both depend on a healthy PostgreSQL).
  \item \code{craftalism-dashboard} and \code{craftalism-minecraft} start once the API is healthy.
\end{enumerate}

\section{Known Limitations}

\begin{itemize}
  \item Image tags in \code{docker-compose.yml} reference \code{latest}, making deployments non-deterministic and non-reproducible across environments.
  \item No TLS termination or reverse proxy is configured; all services are exposed directly on host ports over plain HTTP.
  \item No \code{generate-keys.sh} script exists despite being referenced in \code{env.example}. RSA keys must be generated manually.
  \item No automated health verification or smoke test suite runs post-startup.
\end{itemize}

\section{Roadmap}

\begin{itemize}
  \item Pin all image tags to specific digest or version strings for reproducible deployments.
  \item Add a \code{generate-keys.sh} script for RSA key pair generation and formatting.
  \item Provide an example reverse proxy configuration with TLS termination (Nginx, Caddy, or Traefik).
  \item Add automated post-startup smoke tests that verify all service health endpoints.
\end{itemize}
